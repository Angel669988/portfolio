00 第一性原理 面试开场能定调的一句
传统产品里，PRD 是产品定义的最高形式；AI 产品里，评测集才是。因为"好"是什么，只有当它被打分，才真正存在——没有评测集的 AI 需求，等于没定义。
01 三层评测体系 越往下越接近真实价值，也越难
能力 ───→ 任务 ───→ 体验　｜　离用户价值：远 → 近　｜　成本：低 → 高
① 能力评测
测：单点做得对不对
法：输入-期望输出算准确率
例：意图识别、字段提取准确率
✓ 我做过
② 任务评测
测：端到端任务质量
法：LLM-as-Judge 或人工
例：生成简历整体质量
✓ 我做过
③ 体验评测
测：用户是否真受益
法：行为指标 / A/B / 留存
例：取走率、编辑率
△ 定义+观测行为指标，非严格 A/B
关键认知：不要只做第一层。准确率高 ≠ 用户满意。高阶标志是推到第三层，并关注层与层之间的错位——准确率高但编辑率也高 = 测的"对"不是用户要的"对"。

03 主观质量怎么量化 人工是我做的，LLM-as-Judge 是认知
我做过
人工评测
最准但慢贵。我的做法：规范+打分档位+锚点示例，易偏差样本双人交叉验证一致性。做金标准。
认知
LLM-as-Judge
快、可规模化。三个坑：位置偏见 / 自我偏好 / 打分漂移。核心：Judge 必须先被 judge（人工金标准校准）
认知
规则 / 自动指标
最快最稳，只能测"形"（长度、格式、关键词），测不了"质"
组合而非三选一：规则测硬指标 · Judge 测大批量主观 · 人工做金标准和校准。｜ 我的延伸话术："如果要规模化，我的思路是用现有人工金标准去校准一个 Judge"——用"如果…我的思路是"，不夸大成做过。
04 线上线下对齐 三根因里我解决了两个
已解决
根因① 分布不一致
评测集 query 分布≠线上。我的解法：线上 query 建集 + 分布校准。
认知
根因② 指标不一致
离线准确率 ≠ 线上留存/采用。应选与北极星相关性高的代理指标。
已对抗
根因③ 静态过期
评测集会过期。我的 bad case 周回流正是对抗这个。
进阶认知：离线 ≠ 线上对立，是漏斗。离线（快便宜）高效止损拦掉明显的坏 → 灰度/A/B（慢贵真实）验证价值 → 全量+监控抓长尾。离线作用是"高效止损"，把线上实验资源留给值得验证的版本。

06 "你怎么看 AI 产品评测" · 高度回答框架
① 本质是产品定义
没评测集 = 没定义，"好"必须被打分才存在
② 要分层要对齐
三层越往下越真实；离线线上是漏斗不是对立；分布要匹配
③ 最难的是定义不是技术
Judge 解决规模，但定义好/校准/维护是 PM 的活
最后落到我自己的实践：用线上真实分布校准评测集 + bad case 周回流让评测集持续生长。——认知有高度，实践有落地，两头都占。
